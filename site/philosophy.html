我们不是要教你
省钱，
而是要你清醒。

在这个 AI 狂飙的时代，「用得多」正在悄悄替代「产得好」。STOI 想做的，是让这种无意识烧钱变得可见、可度量、可行动。

「最优秀的工程师，不是写了最多 prompt 的人，而是让每一个被消费的 token 都产生价值的人。」

STOI 这个名字

Shit Token On Investment。它听起来粗鲁，甚至有点冒犯。但正是这种冒犯感，让我们在面对一张天价账单时无法继续自欺欺人。

含屎量越高，说明你的对话里充斥了越多「付了费但毫无价值」的 token：被动态字段破坏的缓存、用户根本没有采纳的冗长建议、已经完成任务后的冗余总结、以及淹没在上下文中间的迷失信息。

我们不希望你因为羞耻感而回避这个数字。相反，我们希望这个数字成为你改进的锚点。

01

可见性先于优化

在你知道 cache miss 来自时间戳之前，所有的「优化 prompt」都是瞎猜。STOI 的第一使命是诊断，而不是建议。

02

可执行胜过正确

「保持对话精简」是一句正确的废话。「在第 8 轮执行 /compact，并将时间戳移出 system prompt」才是可执行的建议。

03

工具应该隐身

最好的审计师不是那个在会议上大声喧哗的人，而是那个让你在下一次会议前已经把问题修好的人。STOI 是背景里的刻薄会计。

我们的设计信条

STOI 的官网没有使用复杂的构建工具，也没有堆砌功能列表。我们希望它像 flomo 或印象笔记一样：简洁、有留白、有手写的温度，同时不失对信息的尊重。

卡片式叙事

每一个理念都被封装在一张独立的卡片中，像笔记一样可以随时被提取、回忆和分享。

留白即强调

我们不害怕空白。空白让重要的句子变得响亮，让用户的注意力集中在真正值得思考的事情上。

自嘲是真诚

页脚那句「这个 STOI 本身也是个 SHIT」不是玩笑。承认自己并不完美，是建立信任最快的方式。

成为清醒的使用者

你不需要立刻同意我们的所有观点。只需要尝试一次：分析你最近的 Claude Code session，看看有多少 token 是被浪费的。
