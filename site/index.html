System Prompt 中一个字符的变动（时间戳从 14:32:15 变成 14:32:16），就足以让 Anthropic 的精确前缀匹配完全失效。你支付的是 100% 的价格，换来的却是 0% 的缓存收益。

PHILOSOPHY

三分层内核

不给你模糊的「好/坏」评分，而是将效率拆成三个可观测、可干预的维度

01

Cache Efficiency

前缀稳定性决定 KV Cache 命中率。命中一次，读取价格仅为正常的 10%。STOI 会帮你找出破坏精确前缀匹配的动态字段。

02

Feedback Validity

输出再长，如果用户没有采纳（没有改文件、没有执行命令），就是无效 token。我们用规则 + 可选 LLM 判断标注每一轮的有效性。

03

Cost Breakdown

将 cache 命中/未命中的价格差异、有效/无效输出占比，换算成实际美元。你会看到自己的「真实成本」和「若无缓存时的成本」。

METHODOLOGY

四层浪费定位法

像外科医生一样，逐层定位对话里的 token 浪费

L1

输入层

JSON 缩进、Markdown 装饰符、无意义换行。

L2

结果层

重复 context、过长的 grep 结果。

L3

结构层

时间戳、UUID、PID 导致 cache miss。

L4

输出层

Yapping、过度解释、冗余总结。
