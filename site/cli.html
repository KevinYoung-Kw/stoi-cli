CLI 参考

一行命令安装，即刻开始审计你的 Token 效率。支持 uv、pipx 和普通 pip。

1 推荐

快速安装

curl -fsSL https://raw.githubusercontent.com/KevinYoung-Kw/stoi-cli/main/install.sh | bash
2 uv

uv 安装（最快，自动管理 PATH）

uv tool install stoi-cli
3 pipx

pipx 安装（自动链接到 ~/.local/bin）

pipx install stoi-cli

安装完成后，运行 stoi help 验证是否可用。如果提示 command not found，请把 ~/.local/bin 加入 PATH 并重新加载终端配置。

CLI 命令速查

所有命令都可以通过 stoi <command> 直接调用，或进入交互式 REPL 后使用 /<command>。

stoi report

分析最新 session，输出含屎量报告与缓存命中率。

stoi report --html

生成可交互的 HTML 报告并在浏览器打开。

stoi dashboard

启动深度分析面板，含对话链条与逐轮 LLM 解读。

stoi start

启动实时监控代理，在后台追踪当前会话的 token 效率。

stoi compare

选择两个 session 做 before/after 对比，直观展示优化效果。

stoi config

进入引导式配置，设置 API key、模型、语音播报等选项。

stoi help

显示所有可用命令的简要说明。

REPL 交互式命令

运行 stoi（不加参数）进入交互式 REPL。这是日常使用最频繁的模式，你可以在分析完一个 session 后立即切换到另一个，或实时查看全局统计。

/report

当前 session 的含屎量分析与输出质量简报。

/dashboard

生成并打开 HTML 分析面板。

/blame

粘贴 System Prompt，定位可能造成 cache miss 的字段。

/mcp

一键配置 MCP，让 Claude Code 直接调用 STOI 工具。

/overview

全局效率报告，汇总所有历史 session 的 token 使用情况。

/sessions

切换要分析的 session（claude / opencode / gemini）。

/insights

基于 LLM 生成深度优化建议（需要配置 API key）。

Ctrl + D

退出 REPL。

MCP 集成

STOI 同时也是一个 MCP Server。如果你把 stoi 接入 Claude Code，Claude 可以直接调用以下工具来分析自己的效率。在 REPL 中输入 /mcp 即可自动完成配置。

  • stoi_latest — 获取最近一次 session 的摘要
  • stoi_report — 对指定 session 做完整分析
  • stoi_insights — 返回基于知识库的深度建议
  • stoi_overview — 全局 token 效率统计
  • stoi_blame — 扫描 System Prompt 中的 cache-busting 模式

准备好开始了吗？

安装后，试试 stoi report，看看你最新一轮 Claude Code 会话的「含屎量」是多少。
