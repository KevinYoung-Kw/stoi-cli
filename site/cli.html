分析原理 CLI 参考 Claude Skill

CLI 参考

说实话，这就是个 Python 脚本包。几行命令扔进去，它就能帮你扒开 Claude Code 的账单，看看有多少 token 是在交智商税。

1 推荐

快速安装

curl -fsSL https://raw.githubusercontent.com/KevinYoung-Kw/stoi-cli/main/install.sh | bash
2 uv

uv 安装（最快）

uv tool install stoi-cli
3 pipx

pipx 安装

pipx install stoi-cli

装完之后运行 stoi help 看看能不能响。如果提示 command not found，请把 ~/.local/bin 加到 PATH 里。这是基础常识，不是 bug。

CLI 命令速查

所有命令都可以通过 stoi <command> 直接调用，或者进 REPL 后用 /<command>。本质上就是python日志解析器加上一个帮腔作势的终端彩打脚本在帮你算账。

stoi report

分析最新 session，告诉你含屎量和缓存命中率。最简单的查账入口。

stoi report --html

生成一个花花绿绿的 HTML 报告并自动打开浏览器。适合截图发朋友圈装逼。

stoi dashboard

启动本地服务，给你一个带逐轮分析的可视化面板。比终端体面一点。

stoi start

启动实时监控代理，在你下一段对话开始前就埋伏好。有点 stalker，但有用。

stoi compare

选两个 session 做 before/after 对比。直观地证明你优化（或者搞砸）了多少。

stoi config

配置 LLM Provider、API key、模型等参数。填错就报错，很诚实。

stoi help

显示所有可用命令的简要说明。迷路时的紧急出口。

REPL 交互式命令

运行 stoi（不加参数）进入一个丑但好用的交互式 REPL。这是日常使用最频繁的模式，你可以边聊边查，不用每次都重新敲命令。

/report

当前 session 的含屎量简报。适合对话中途突然想查账。

/dashboard

就地生成并打开 HTML 分析面板。REPL 里一键装逼。

/blame

粘贴 System Prompt，定位导致 cache miss 的嫌疑字段。甩锅专用。

/mcp

一键把 STOI 塞进 Claude Code 当 Skill。配置粗暴但有效。

/overview

全局效率报告，汇总所有历史 session。看看你一共烧了多少冤枉钱。

/sessions

切换要分析的 session（claude / opencode / gemini）。多平台渣男追踪器。

/insights

基于 LLM 生成深度优化建议。需要配置 API key，天下没有免费的建议。

Ctrl + D

退出 REPL。去干活吧，别光查了。

MCP 集成

STOI 同时也是一个 MCP Server。如果你把 stoi 接入 Claude Code，Claude 可以直接调用以下工具来分析自己的效率。在 REPL 里敲 /mcp 就能完成配置。说白了就是把 CLI 的功能以 JSON-RPC 的形式暴露出去，方便 Claude 自己查自己。

  • stoi_latest — 获取最近一次 session 摘要
  • stoi_report — 对指定 session 做完整分析
  • stoi_insights — 基于知识库的优化建议
  • stoi_overview — 全局 token 效率统计
  • stoi_blame — 扫描 System Prompt 里的 cache-busting 模式

准备好开始查账了吗？

安装后试试 stoi report，看看你最新一轮 Claude Code 对话的「含屎量」是多少。数字不好看别怪我们，怪你的 Prompt。
